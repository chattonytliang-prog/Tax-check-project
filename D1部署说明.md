# Cloudflare Pages + Functions + D1 部署说明

## 一、当前架构

当前项目采用：

```text
Cloudflare Pages        前端静态站点
Pages Functions         后端 API
Cloudflare D1           企业档案、报告、风险结果数据库
```

后端 API 文件位于：

```text
functions/api/
```

D1 建表 SQL 位于：

```text
migrations/0001_initial.sql
migrations/0002_auth.sql
migrations/0003_user_data_isolation.sql
migrations/0004_admin.sql
```

## 二、Cloudflare 控制台操作步骤

### 1. 创建 D1 数据库

进入 Cloudflare 控制台：

```text
Workers 和 Pages -> D1 SQL Database -> Create database
```

建议数据库名：

```text
tax-check-db
```

### 2. 创建数据表

进入刚创建的 D1 数据库，找到 Console / Query 页面。

将 `migrations/0001_initial.sql` 里的 SQL 全部复制进去执行。

执行后应创建这些表：

```text
clients
reports
risk_results
risk_rules
```

如果要启用注册登录，再执行 `migrations/0002_auth.sql`，会创建：

```text
users
sessions
```

如果要让每个测试用户只能看到自己的企业和报告，再执行 `migrations/0003_user_data_isolation.sql`，会给这些表增加归属字段：

```text
clients.owner_user_id
reports.owner_user_id
```

如果要启用管理员后台和代入用户功能，再执行 `migrations/0004_admin.sql`，会增加：

```text
users.role
users.disabled_at
sessions.actor_user_id
audit_logs
```

然后把你的管理账号设为 admin，例如：

```sql
UPDATE users SET role = 'admin' WHERE username = 'test1';
```

### 3. 给 Pages 项目绑定 D1

进入 Pages 项目：

```text
tax-check-project -> Settings -> Functions -> D1 database bindings
```

添加绑定：

```text
Variable name: DB
D1 database: tax-check-db
```

注意变量名必须是：

```text
DB
```

因为后端代码使用 `env.DB` 访问数据库。

### 4. 重新部署 Pages

绑定 D1 后，回到 Deployments 页面，触发一次新部署。

可以在本地推一个空提交，也可以在 Cloudflare 页面点重新部署最新提交。

### 5. 验证 API

部署完成后访问：

```text
https://tax-check-project.pages.dev/api/health
```

如果成功，应看到类似：

```json
{
  "ok": true,
  "database": "connected",
  "clients": 0
}
```

如果失败，常见原因：

- 没有绑定 D1；
- 绑定变量名不是 `DB`；
- 没有执行建表 SQL；
- 仍在访问旧部署。

## 四、提前注意

### 1. 第一次部署后不一定马上持久化

如果 D1 没有绑定或表没创建，前端会自动进入演示数据模式。

页面顶部会显示：

```text
当前为演示数据模式：请在 Cloudflare Pages 绑定 D1 数据库后使用持久化能力。
```

### 2. 当前规则检测仍在前端

第一版只是把企业档案和报告保存到 D1。

风险规则仍然在前端代码中执行，这是为了保持 MVP 简单。

后续可以把 `risk_rules` 表做成真正的规则后台。

### 3. 当前账号系统是测试期门禁

当前已经支持用户名 + 密码注册登录，密码会哈希后保存到 D1，不会明文保存。

执行 `0003_user_data_isolation.sql` 后，企业档案和报告会按当前登录用户隔离。

正式使用前必须补：

- 组织/租户隔离；
- 操作日志；
- 数据权限；
- 敏感信息保护。

### 4. 报告 Word 仍在前端导出

当前 Word 导出仍由浏览器生成。

后续如果需要正式报告管理，可以把生成后的报告文件存到 Cloudflare R2。
