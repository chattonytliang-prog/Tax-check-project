# 合耀税务风控工作台产品备忘录

更新时间：2026-06-22

## 一、产品简介

合耀税务风控工作台是一款面向企业内部财税、税务、风控和经营管理团队的 AI 税务风险体检工具。

产品通过录入企业基础信息、经营数据、开票数据、人员社保数据、费用情况、优惠政策适用情况等财税画像信息，自动匹配内置税务风险规则，生成风险等级、触发原因、整改建议、需补充资料清单，并输出可复核、可流转的税务风险体检报告。

当前版本是 MVP，重点验证一条核心链路：

```text
企业画像录入 -> 风险规则检测 -> 风险结果展示 -> 报告生成 -> Word 导出
```

产品暂不定位为自动报税、自动做账或税务筹划决策系统，而是定位为：

```text
税务风险识别与报告生成工作台
```

## 二、目标用户

第一阶段适合以下用户：

- 企业内部财务负责人
- 企业税务负责人
- 财税风控人员
- 集团总部税务管理团队
- 税务顾问或专业服务人员
- 需要定期做税务风险体检的小微企业管理者

后续可以扩展到：

- 多企业集团税务管理
- 财税咨询机构
- 会计师事务所/税务师事务所
- 小微企业 SaaS 服务场景

## 三、核心功能

当前 MVP 已包含：

- 合耀科技品牌化登录页和工作台 UI
- 企业档案列表
- 企业财税画像录入
- 30 条税务风险测试规则
- 风险等级识别：高 / 中 / 低
- 风险触发原因展示
- 整改建议和需补充资料提示
- 报告正文生成
- Word 格式导出
- 规则库展示

## 四、当前产品优势

### 1. 切入点清晰

产品没有一开始做完整财税系统，而是先聚焦“税务风险体检报告”这一高价值、低复杂度场景。

这个切入点更容易让用户理解价值，也更容易做 MVP 验证。

### 2. 专业壁垒明显

内置规则来自公开法规、税务风险案例和财税实务经验，方向上避开了通用 AI 问答工具的泛化问题。

产品核心不是“让大模型自由回答税务问题”，而是用结构化规则先做风险识别，再用报告承载结果，风险边界更清楚。

### 3. 交付物明确

很多 AI 工具只提供聊天结果，本产品直接生成《税务风险体检报告》。

报告可以用于：

- 内部复核
- 风险沟通
- 整改跟踪
- 管理层汇报
- 后续专业人员复核

### 4. 技术落地轻

当前版本是前端 MVP，部署在 Cloudflare Pages，成本低、迭代快、便于演示。

后续可以自然接入：

- Cloudflare Pages Functions
- Cloudflare D1 数据库
- Cloudflare R2 文件存储
- AI API

### 5. 品牌感已初步建立

界面已结合合耀科技图标，使用深色科技侧边栏、青蓝发光主色和工作台式布局，让产品看起来不像临时 Demo，而更像公司正式产品。

## 五、当前产品劣势

### 1. 数据暂未持久化

当前企业数据和报告数据主要保存在前端内存中，刷新后会回到演示状态。

这是下一阶段必须优先解决的问题。

### 2. 风险规则仍是测试规则

当前 30 条规则适合 MVP 演示和产品验证，但还不是正式税务风控规则库。

后续需要继续补充：

- 行业细分规则
- 地区政策差异
- 规则阈值
- 政策依据版本
- 规则启停和优先级
- 专业人员复核流程

### 3. 尚未接入真实 AI 能力

目前报告生成主要基于模板和规则结果，还没有接入大模型 API。

后续 AI 应主要用于：

- 报告语言润色
- 风险摘要生成
- 整改建议改写
- 资料清单归纳
- 管理层简版说明生成

AI 不应直接承担最终税务判断。

### 4. 尚无权限体系

当前没有真实登录、用户、组织、角色和权限控制。

正式产品需要支持：

- 用户账号
- 企业组织
- 角色权限
- 操作日志
- 数据隔离
- 敏感信息保护

### 5. 尚未接入真实财税数据源

当前主要依赖手工录入。

后续如果要提升实用性，需要考虑导入：

- Excel 表格
- 发票汇总
- 银行流水
- 工资社保表
- 申报表
- 平台收入数据
- 账套导出数据

## 六、适合优先验证的问题

第一阶段不应验证“大而全的财税系统”，而应验证以下问题：

1. 用户是否认可“税务风险体检报告”这个交付物？
2. 当前 30 条规则中，哪些最有业务价值？
3. 报告表达是否足够专业、谨慎、可复核？
4. 企业内部团队是否愿意定期使用这个工具？
5. 哪些字段是用户愿意填写的，哪些字段太重？
6. 报告是否能帮助推动整改或内部沟通？
7. 用户是否愿意为数据保存、报告模板、规则库升级付费？

## 七、下一阶段建议

优先级从高到低：

### P0：接入后端和数据库

目标：

```text
新建企业后刷新页面，数据仍然存在。
生成报告后刷新页面，报告仍然存在。
```

建议技术：

- Cloudflare Pages Functions 做 API
- Cloudflare D1 做数据库

### P1：规则库后台化

让规则从代码中迁移到数据库，支持后台维护：

- 新增规则
- 修改规则
- 启用/停用规则
- 绑定政策依据
- 设置风险等级
- 设置适用行业和纳税人类型

### P2：报告模板升级

将报告做成更正式的交付物：

- 封面
- 摘要页
- 风险矩阵
- 风险明细
- 整改清单
- 需补充资料
- 免责声明
- Word/PDF 双格式导出

### P3：接入 AI

AI 只做辅助表达，不做最终判断：

- 风险摘要
- 整改建议改写
- 管理层简版
- 专业版/通俗版报告切换

### P4：数据导入

支持 Excel/CSV 导入，提高真实使用效率。

## 八、产品边界声明

本产品应坚持以下边界：

- 不承诺节税结果
- 不替代税务师或专业人员判断
- 不直接生成最终税务处理意见
- 不做无依据筹划建议
- 不包装成税务机关官方风控模型
- 不声称使用“内部指标”

推荐表达：

```text
本系统基于公开法规、公开案例及企业提供资料进行辅助分析，输出内容仅供税务风险管理和内部复核参考。具体税务处理应结合完整原始资料、适用地区口径及最新政策，由专业人员进一步确认。
```

## 九、一句话定位

合耀税务风控工作台是一款将税务专业经验沉淀为规则和报告的内部风控工具，帮助企业快速识别涉税风险、形成整改线索，并生成可复核、可流转的税务风险体检报告。
